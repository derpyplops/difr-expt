# Experiment log: fp-emulation
Started 2026-05-12. See experiments/fp-emulation/plan.md and docs/plans/fp-emulation-published-teachers.md.

- 2026-05-12: Scaffolding — built directory, operational plan, and shell scripts (`setup_gpu.sh`, `cache_all_prompts.sh`, `run_phase0_smoke.sh`, `run_phase2_training.sh`). Added `--teacher-source published` path to `src/difr_expt/train_emulate.py` (~50 LOC) so teacher can be loaded from a published HF checkpoint (compressed_tensors for fp8, fp_quant for NVFP4) instead of always being a `LowPrecisionLinear`-patched copy of the base. Added `--use-8bit-adamw` (bitsandbytes) and `--grad-checkpointing` flags — both needed to fit 8B models on H100 80GB (memory budget: teacher dequant ~8GB + fp32 shadows ~32GB + 8bit AdamW state ~16GB + activations ~5GB ≈ 60GB, comfortably under 80). Also fixed deprecated `torch_dtype=` → `dtype=` in `from_pretrained` calls. Scripts include ntfy hooks to `claude-jon-alerts` for per-run start/end + failure notifications.
